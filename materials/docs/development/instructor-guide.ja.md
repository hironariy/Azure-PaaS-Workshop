# インストラクターガイド

講師向けの詳細な進行メモはリポジトリ内の `docs/instructor-guide.ja.md` を参照してください。

Cloud Shell 専用ポータルでは、受講者本線を次の順序に揃えています。

1. [受講者クイックスタート](../learner/cloud-shell-quickstart.ja.html)
2. [Day 0: 事前準備](../learner/day-0-prerequisites.ja.html)
3. [Day 0: Entra ID と認証設定](../learner/day-0-entra-id.ja.html)
4. [Day 1: PaaS インフラをデプロイ](../learner/day-1-deploy-infrastructure.ja.html)
5. [Day 1: バックエンドをデプロイ](../learner/day-1-deploy-backend.ja.html)
6. [Day 1: フロントエンドをデプロイ](../learner/day-1-deploy-frontend.ja.html)
7. [Day 1: アプリを検証](../learner/day-1-validation.ja.html)
8. [Day 2: 監視と運用](../learner/day-2-operations.ja.html)
9. [Day 2: 信頼性と復旧](../learner/day-2-reliability.ja.html)
10. [Cleanup](../learner/cleanup.ja.html)

テナントで Entra ID app registration 作成が制限されている場合は、講師が事前に Backend API / Frontend SPA の app registration を作成し、受講者へ Tenant ID と Client ID を配布してください。
